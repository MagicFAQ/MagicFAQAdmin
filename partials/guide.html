User Guide

MagicFAQ is our in-house FAQ system. Visitors can search using conversational English to find Questions and Answers. The system learns from use, incorporating user feedback and behavior to improve its recommendations.

Websites

We have a wordpress plugin, which makes it easy to add MagicFAQ to most UW websites. You can see a primary example of this on the OUR website. MagicFAQ is not supported on any legacy Drupal websites. We can set up multiple wordpress sites to use the same instance of MagicFAQ, showing the same FAQ, or separate them completely.

Visitors are greeted with a list of the top questions as generated by the system. This list may change as visitor searches change.





When a visitor clicks on a question, it will expand to show the associated answer, as well as a series of options to provide feedback to the system. Visitor feedback is incorporated back into the search algorithm, allowing MagicFAQ to learn over time which searches to associate more strongly with which Q/A pairs.

Having clicked on a Question, the user sees an answer and feedback options.




Administration

Once we have a copy of MagicFAQ installed and configured for you, all administrative tasks can be done though a dedicated administrative web app. Each instance of MagicFAQ will have it's own control panel.

Administrators are managed by NetID and UW Groups. This way your department can handle access permissions without having to generate a help ticket. If you don't know how to use the UW groups service, Enrollment Information Services (EIS) is always available to assist.

Questions

Upon opening the Administrative panel, you'll be greeted by a list of all active questions. You can also see the main navigation menu across the top of the screen.



Clicking on a question will take you to the detail screen. From here you can add recommended terms, see recent searches which have yielded this Q/A pair, and edit the question itself.

Recent Searches and Recommended Search Terms may look a little strange the first time you see them. The system automatically strips out connector words, and simplifies the text down to a very sparse form of language. This reduced form is best for the search algorithm which powers MagicFAQ.



Adding New Questions

Adding a new Question / Answer pair is a simple process. Click the "Create a Question" menu button at the top of the screen and you'll be given a basic form to fill out. Once the question is created, it will immediately start showing up in search results.


Adding Recommended Search Terms

If you scroll to the bottom on a Question Detail screen, you'll see a box for entering new recommendations. You can also tag recent search terms, and add them directly to the recommended list. Visitors to the website can also recommend search terms, so you will see this list grow even if no one in your office is actively managing it.



Testing the FAQ

Once you have some questions and answers entered into the system, it can be helpful to try out some searches and make sure the correct questions are showing up. This can be done from the "Test FAQ" page, or from the public website. The Test page does not incorporate any feedback.
